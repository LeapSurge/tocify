# Tocify 三阶段向导 UX 设计

## 背景与目标
- 目标：提升“新用户一次走通”的成功率。
- 范围：重构界面结构与关键交互，不做大幅视觉重绘。
- 用户路径：`上传与预处理 -> 生成与编辑 -> 后处理与导出`。

## 信息架构
- Stage 1 预处理
  - 左栏：说明文本、API 设置、页码范围、开始按钮。
  - 右栏：PDF 预览（网格模式）。
  - 规则：未上传文件时开始按钮禁用并给出原因。
- Stage 2 生成与编辑
  - 左栏：目录编辑区（文本/拖拽）。
  - 右栏：单页预览与定位联动。
  - 导航：上一步、下一步。
- Stage 3 后处理与导出
  - 左栏：后处理设置（编号、实体目录页、页码标签等）。
  - 右栏：最终预览。
  - 导航：上一步、导出按钮。

## 交互规则
- 全局仅保留一个“上传新文件”按钮，作为重启流程入口。
- 主操作禁用时必须显示单一原因文案。
- API 设置支持未保存提示，避免“看似已生效”的误解。
- 文本 AI 格式化失败时，loading 状态必须可靠回收。

## 状态流转
- `stage = 1 | 2 | 3`
- `1 -> 2`：生成成功或已有可编辑目录。
- `2 -> 3`：目录非空。
- `3 -> 2`：允许回退继续编辑。
- 重新上传文件：回到 Stage 1。

## 验收标准
- 新用户能在首屏理解“下一步做什么”。
- 关键按钮禁用时都有明确原因。
- API 配置变更不会出现“未生效但无提示”。
- 文本 AI Format 失败后，不会卡在处理中。
